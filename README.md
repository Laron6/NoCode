# Календарь событий

Приложение для планирования событий: добавление от конкретного пользователя, редактирование, удаление, уведомления в браузер. Данные сохраняются в localStorage.

Автор: Bogdan Matrosov  
Год: 2026

## Запуск
- Дважды кликнуть index.html  
- Тесты: `node --test calendar.test.js`

## Сценарии тестирования

### Тест 1. Добавление пользователя и события
**Цель**: Создание пользователя и события от него  
**Предпосылки**: Страница открыта  
**Шаги**:  
1. Нажать "+ Новый пользователь" и ввести имя  
2. Выбрать пользователя в селекте  
3. Заполнить название, дату, время, описание  
4. Нажать "Добавить событие"  

**Ожидаемый результат**: Событие появляется в списке с именем создателя, сохраняется после перезагрузки  
**Фактический результат**: Событие добавлено от выбранного пользователя, имя отображается корректно

### Тест 2. Редактирование события
**Цель**: Изменение данных события  
**Шаги**:  
1. Добавить событие  
2. Нажать "Редактировать" на карточке  
3. Изменить данные и сохранить  

**Ожидаемый результат**: Событие обновляется, изменения сохраняются  
**Фактический результат**: Название, дата, время и описание успешно изменились

### Тест 3. Удаление события
**Цель**: Удаление записи из календаря  
**Шаги**:  
1. Добавить событие  
2. Нажать "Удалить" на карточке  
3. Подтвердить  

**Ожидаемый результат**: Событие исчезает из списка  
**Фактический результат**: Событие удалено, список обновлён

### Тест 4. Уведомления в браузере
**Цель**: Проверка уведомлений о предстоящих событиях  
**Шаги**:  
1. Разрешить уведомления в браузере (если ещё не разрешены)  
2. Добавить событие на ближайшее время  
3. Подождать или перезагрузить страницу  

**Ожидаемый результат**: Появляется системное уведомление о предстоящем событии  
**Фактический результат**: Уведомление работает при разрешении

### Тест 5. Сохранение после перезагрузки
**Цель**: Данные не теряются  
**Шаги**:  
1. Добавить пользователей и события  
2. Перезагрузить страницу  

**Ожидаемый результат**: Все пользователи и события на месте, имена создателей сохраняются  
**Фактический результат**: Всё сохранилось, пользователи и события отображаются корректно

### Тест 6. Сортировка событий по времени
**Цель**: События отображаются в хронологическом порядке  
**Шаги**:  
1. Добавить события в случайном порядке дат  
2. Посмотреть список  

**Ожидаемый результат**: События отсортированы от ближайших к дальним  
**Фактический результат**: Список всегда в правильном порядке по времени

## Очистка данных
Очистка через `clearAll()` в коде или перезапись localStorage → все пользователи и события удаляются.  
После очистки списки пустые.

Все основные сценарии протестированы вручную и автоматически (unit-тесты проходят успешно). Приложение полностью работоспособно.
